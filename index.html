🛠️ 2D Terminal Graphics Engine

A menu-driven, state-managed 2D graphics editor written entirely in C. It utilizes a custom Display List architecture to allow dynamic modification and deletion of geometric objects on an ASCII canvas.

Core Features

  • Mathematical Rasterization: Implements Bresenham's Line Algorithm and Midpoint Circle Algorithm from scratch.
  • State Management: Uses a `struct` based Display List to track Object IDs, rather than destructively drawing to an array.
  • Dynamic Canvas: Add, Delete, and Modify geometries (Lines, Rectangles, Circles, Triangles) in real-time.
  • Aesthetic CLI: Standard ANSI escape sequences map out a modern, color-coded terminal user interface.

Quick Start Guide

To run this project locally, ensure you have a C compiler (like GCC) installed, then run the following commands in your terminal:

$ gcc editor.c -o editor -lm
$ ./editor

Project Architecture

The system operates on an infinite game loop. During each frame, the canvas is wiped clean to _ characters. The engine iterates through the active `display_list` and recalculates the coordinates for all shapes, painting them with * characters before printing the final buffer to standard output.
